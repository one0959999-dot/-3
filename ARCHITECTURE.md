# 실전 운용 아키텍처 — 백테스트 데이터·로직·봇+AI 역할 (2026-06-24)

전략 원칙: **알고리즘 主 · AI 補 · 백테스트가 단일 기준(single source of truth)**

## 1. 백테스트 데이터 활용법
`backtest_trade_signals` (KR 60.8만 + US 149만 = 209만건). 라이브가 이 데이터를 4가지로 씀:

| 용도 | 함수 | 데이터 |
|---|---|---|
| **진입 게이트** | entry_engine.evaluate | 국면×신호별 win10·win20 (idx_bts_mode_phase 인덱스) |
| **예상수익 표시·정렬** | signal_forecast.estimate_candidate_return | 신호별 avg max_gain (없으면 국면평균 폴백) |
| **청산선(표시=실제)** | _capture_bt_levels → 관리루프 | 손절=avg max_drawdown, 익절=avg max_gain |
| **AI 판단 근거** | get_forecast → ai_finetune_decision | 승률·목표·손절·표본을 프롬프트에 주입(신규) |

→ 데이터가 쌓일수록 자동으로 정확해짐(재학습 불필요).

## 2. 진입 로직/알고리즘 (게이트 순서)
```
스크리너(거래량·테마·모멘텀)              ← 알고리즘
   ↓ 후보
exp_return 산출·정렬                       ← 백테스트
   ↓
AI 종합 후보선정(국면·섹터·정세)          ← AI 補
   ↓
점수제 calculate_entry_score ≥ threshold  ← 알고리즘(10지표)
   ↓
앙상블 게이트: engine_buy(신호+win10+win20) ← 알고리즘+백테스트  ★핵심
   ↓
AI 최종 승인(veto) — 백테스트 통계 보고 판단 ← AI 補(악재·모순만 거부)
   ↓
_buy_order → killswitch 게이트 → 체결
```
**알고리즘이 "산다"를 결정하고, AI는 명백한 악재/모순일 때만 막는 구조.** (앙상블 대결서 +4.5% 최고)

## 3. 청산 로직 (다층 안전망, 먼저 걸리는 게 발동)
1. **killswitch L2** (포트폴리오 -20%) → 전량청산  ← 최우선
2. **백테스트 손절/익절선** (표시=실제)            ← 백테스트
3. 장초 급락손절 / ATR 트레일링·하드손절          ← 알고리즘
4. 과열청산(테마·RSI 단계적)                       ← 알고리즘
5. 1·2차 분할익절 (AI 검토 가능)                   ← 알고리즘+AI
6. 스크리너 제외 손절                              ← 알고리즘

## 4. 봇 + AI 역할 분담
| | 알고리즘(봇) | AI(Gemini/Claude) |
|---|---|---|
| 역할 | **결정권자** | **자문·veto·후보선정** |
| 진입 | 점수+앙상블(백테스트)로 매수 결정 | 악재/모순 시 거부, 시장국면 가점 |
| 후보 | 스크리너 정량 선별 | 국면·섹터·정세 종합 선정 |
| 청산 | 손절/익절선 자동 | 분할익절 타이밍 자문 |
| 실패시 | — | API 실패→자동승인(fail-open, 봇 계속) |

**AI는 백테스트 통계를 받아 판단** → 알고리즘과 같은 데이터로 사고(충돌 방지). AI가 죽어도 알고리즘은 작동.

## 5. 안전장치
- 포트폴리오 2단계 killswitch (-10% 신규중단 / -20% 전량청산)
- per-ticker 예외격리(한 종목 오류가 전체 안 막음) — KR·US 동일
- 거래비용 반영(매도세+수수료), 룩어헤드/생존편향 보정(상폐수집)
- 상태 영속(재시작해도 손절선·포지션 유지)
