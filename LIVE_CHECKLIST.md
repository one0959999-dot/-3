# 실전 검증 체크리스트 — "진짜 되는가" 확인용

원칙: 새 기능 추가 ❌. **현재 봇이 의도대로 도는지 관찰**하는 게 목적.
순서: 모의(mock) 며칠 → 소액 실거래 1~2주 → 그다음에야 "실전 가능".

## 0. 가동 전 (1회)
- [ ] **PC 절전/최대절전 끄기** (전원옵션 → 절대 안 함). ← 야간 2회 죽은 원인, 최우선
- [ ] 실/모의 모드 확인: 어떤 토스키(real vs mock)로 봇이 뜨는지 로그에서 확인
- [ ] `python tests/test_core_logic.py` → 31 통과 확인
- [ ] 텔레그램 알림 수신 테스트(봇 시작 메시지 오는지)
- [ ] killswitch 켜짐 확인: `KILLSWITCH_ENABLED=True`

## 1. 첫날 — 기본 생존
- [ ] KR/US 봇 프로세스가 장 시간 내내 안 죽고 유지되는가
- [ ] 잔고/현재가 동기화가 30~60초마다 도는가 (로그)
- [ ] 에러 로그에 반복되는 예외(특히 "위성 매매 오류", "AI 게이트 오류")가 없는가

## 2. 종목 선정 검증
- [ ] 선정된 후보가 텔레그램/로그에 뜨고, **exp_return 높은 순**인가
- [ ] 관리종목/거래정지 종목이 후보/매수에 **안 들어오는가** (위험필터 작동)
- [ ] AI 후보선정 이유가 국면·섹터 근거를 담는가

## 3. 진입 검증 (핵심)
- [ ] 실제 매수 발생 시: 점수 통과 → 앙상블(engine_buy) → AI승인 순서가 로그에 찍히는가
- [ ] 매수 메시지의 **목표/손절선이 백테스트 통계값**과 일치하는가
- [ ] 한 종목이 포트폴리오 30% 넘게 안 사지는가 (집中도)
- [ ] killswitch -10% 도달 시 신규매수가 실제로 멈추는가 (장 급락일 관찰)

## 4. 청산 검증 (핵심)
- [ ] 손절/익절이 **매수 때 띄운 선과 같은 값**에서 작동하는가 (표시=실제)
- [ ] 미체결 지정가가 10분 후 취소되는가 (체결확인 루프)
- [ ] 부분익절(1·2차)이 의도대로 단계적으로 나가는가

## 5. AI·자기개선 검증 (며칠~주 단위)
- [ ] AI 판단이 `ai_decision_log`에 쌓이는가 (`SELECT COUNT(*)`)
- [ ] 5일 후 AI 사후검증 적중률 텔레그램이 오는가
- [ ] 자기개선 자동반영 알림이 오면, `strategy_params_history`에 이력이 남는가
- [ ] 주간 파생통계 재집계 로그가 찍히는가

## 6. 회계 검증
- [ ] 봇 내부 손익 vs 토스 실제 잔고가 크게 어긋나지 않는가
- [ ] 거래비용(수수료·세금) 반영 후 손익이 맞는가

## 🚩 즉시 중단(봇 끄기) 신호
- 같은 종목을 반복 매수/매도(휩쏘 루프)
- 의도치 않은 대량 주문
- killswitch가 안 도는데 손실 -20% 초과
- 잔고와 내부 상태가 크게 불일치

## 검증 후 (그때 비로소)
- 위 항목 대부분 ✅ → 소액 실거래 점진 확대
- 그 후에야 리팩터(파일 세분화)·기능 추가 고려 — 테스트(test_core_logic.py)를 회귀 기준선으로 사용
