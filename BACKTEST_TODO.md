# ⭐ 방법론 결론 (2026-06-26 논의 — 제일 먼저 읽을 것)

"전부 다 검증한다"는 방향은 OK. 문제는 검증이 아니라 "1등 뽑는 기준" 딱 하나뿐.

## 핵심 규칙
- 전 종목 + 전체 기법/경우의 수 검증해도 됨 (원리적으로 안 틀림. 단 단일 PC면 8백만 x 전종목 x 12년 = 며칠~몇주, 클라우드면 비용 문제. "틀림"이 아니라 "비쌈/느림").
- 절대 금지: "과거 수익 1등"을 골라 믿기 → 8백만 돌리면 운으로 1등 한 허상이 수천 개.
- 진짜 순위 기준 = "전 종목에서 + 안 본 기간(OOS)에서도 1등". 시간 둘로 쪼갬: 만들기 2014~2020 / 검증(안 본) 2021~2026. 이 필터 통과한 것만 채택 = 통계적으로 정당.

## 국면조건부(regime-conditional) 설계 — 유력 방향
- 매일 "과거 데이터만" 보고 국면 판정 (미래 훔쳐보기=lookahead 금지, walk-forward).
- 하락장용 베스트 조합 / 상승장용 베스트 조합 따로. (8백만 통째로 X → 국면별 소수 O, 경제적 근거 있음).
- 국면 스위치는 봇 8단계(base/market_phase.py) 그대로 사용 — 이미 검증됨.
- 종결 비교: 적응형(국면조건부) vs 봇단독 vs 단순보유 vs "글로벌 1등 조합". 적응형이 OOS에서 글로벌1등을 이기면 → 국면별로 나누는 게 맞다는 증거.

## 의심 지점 (반드시 백테스트로 깰 것)
- 상승장 "고점매도/저점매수로 주식수 불리기"는 단순보유에 질 가능성 높음 (강불장에선 팔면 더 오름 + 세금/수수료). 사용자 원래 목표라 꼭 확인.
- 봇은 이미 8단계로 국면별 행동 중 → 그 위에 얹으면 복잡도만 늘 수도.

## 지금까지 발견 (하락장 2021-11~2022-12, 에스앤에스텍 101490)
- 봇 단독 +19%(MDD -14%)가 사실상 천장. 22개 기법 어떤 것도 의미있게 못 이김.
- BOLL "+20%(1위)"는 허상: BOLL 보유%가 봇과 동일(40%) = 아무것도 안 바꿨는데 진입일 며칠 어긋나 +1%p. = "아무것도 안 한 게 1위" = 노이즈 순위의 정체.
- CHAND는 MDD를 -14→-13으로 1%p 줄임(보유 34%) 하지만 수익 -2%p trade-off. 한 종목/한 기간이라 미검증.
- 교훈: 22개에서 가짜1등(BOLL) 1개 발생 → 8백만이면 가짜1등 수천 개. brute-force 채점의 위험을 실증함.
- 관련 스크립트: KR/all_methods_backtest.py, KR/synergy_backtest.py, KR/synergy_all_backtest.py, KR/combined_hedge_backtest.py

# 🎯 국면별 실전 타이밍 전술 (저점매집 / 익절 / 돌파)

원칙: 전환점은 "예측" 불가, "확인 후 반응"만 가능. 위 "방법론 결론"의 채점(전 종목 + OOS)을 그대로 적용.
유망순: ① 하락장 저점 분할매수 > ③ 상승장 추세이탈 익절 > ② 횡보.

## ① 하락장 — 저점 "분할매집" (제일 유망)
- 한 점(바닥) 찍기 X → 과매도 "구간"을 분할로 사 모음(평균회귀 근거).
- 후보 신호: RSI 과매도+반등, 거래량 캐피출레이션, 봇 PANIC/RECOVERY 국면, 200MA 이격 과대.
- 백테스트: "확인 후 분할매수" vs "추세반전 확인 후 일괄매수" vs 단순보유.

## ③ 상승장 — "추세이탈 익절" (고점매도 아님)
- 정확한 고점 X → 추세 깨지면 늦게라도 익절(트레일링 스톱).
- 주의: 강불장에선 일찍 팔면 보유에 짐(이미 의심됨) → "역시 들고 가는 게 낫다"를 숫자로 확인하는 의미.
- 백테스트: 트레일링스톱(ATR 배수 여러개) vs 단순보유.

## ② 횡보장 — 방향예측 말고 박스로 접근 (3가지 테스트)
- 방향 예측은 거의 동전(못 함). 잡히는 건 "큰 움직임 임박"(볼린저 스퀴즈/ATR 수축)뿐, 방향은 아님.
- A) 레인지매매: 지지에서 사고 저항에서 팜(박스 깨지면 손절). ★ 사용자의 "고점팔고 저점사서 주식수 불리기"는 강불장이 아니라 여기(횡보/박스권)가 제 집 — 박스권에선 그 스윙이 먹힘.
- B) 스퀴즈→돌파 추종: 수축 감지 후 터지는 방향으로 따라붙기(예측 아님, 반응).
- C) 관망(노출 축소): 횡보엔 안 싸우는 게 최선일 수도 → 베이스라인.
- 백테스트: 횡보 구간에서 A/B/C 중 OOS 최강 찾기.

# 🚀 최종 실행 스펙 (로컬 turnkey — 결정 완료 2026-06-26)

아래 순서대로 실행하면 됨. 결정 끝났으니 로컬은 "확인 + 실행"만.

## 확정 결정
- 검증 범위 = 대표 샘플 먼저 (KR + US 30~50종목, 업종/시총 다양). 잘 나오면 그 다음 전 종목 확대.
- 알고리즘 단위 = 종목별 타이밍 먼저 (포트폴리오/코어-새틀라이트 자금배분은 나중).
- 국면 = 봇 8단계 (base/market_phase.py) 그대로. 매일 과거데이터만 보고 판정(lookahead 금지).
- 기간 = 만들기 2014~2020 / 검증 2021~2026 (검증은 한 번도 안 본 데이터).
- 채점 = (검증기간에 단순보유 이긴 종목 %) x (수익/MDD 중앙값). 둘 다 높은 게 1등.
- 후보는 국면별 5개 이하로 제한 (8백만 조합 폭발 금지).

## 실행 순서
- STEP 0 (관문): lassi.db에 샘플종목 12년치 OHLCV(시/고/저/종) 있는지 확인. 없으면 한 번만 받기(KR=pykrx, US=yfinance). 있으면 그거 사용.
- STEP 1: 대표 샘플 30~50종목 선정(KR 대형/중형/소형 + US 대형/성장/가치 골고루). 목록 파일로 고정.
- STEP 2: 종목마다 매일 봇 8단계 국면 판정(과거만).
- STEP 3: 국면별 후보 메뉴 정의 (각 5개 이하):
    하락장 → 분할매집: RSI과매도+반등 / 봇RECOVERY / 200MA이격과대 (분할 vs 일괄 vs 단순보유)
    상승장 → 추세이탈익절: ATR 트레일링(배수 2/3/4) vs 단순보유
    횡보장 → A)레인지매매 B)스퀴즈→돌파추종 C)관망
- STEP 4: 만들기기간(2014~2020)에서 국면별 베스트 후보 선정.
- STEP 5: 검증기간(2021~2026)에 그 베스트를 walk-forward 적용 → 채점.
- STEP 6: 종결 비교표(종목 가로 집계): 적응형(국면조건부) vs 봇단독 vs 단순보유 vs 글로벌1등조합.
- STEP 7: 적응형이 검증기간에 이기면 → 그 {국면→행동} 룰표 = 알고리즘. 지면 → 봇단독 유지(그것도 유효한 결론).

## 산출물
- {하락장→X, 상승장→Y, 횡보장→Z} 룰표 + 검증 성적표.
- 통과 기준: 적응형이 "검증기간 + 다수 종목"에서 봇단독/글로벌1등을 이김. (한 종목/한 기간 1등은 무시 = 운빨)

# 백테스트 로드맵 — 로컬 Claude 작업 우선순위 (TOP PRIORITY)

이 파일을 가장 먼저 읽고 작업할 것.
목표: "국면 파악 + 대응 매매"를 봇 기존 방식과 비교 검증해서, 데이터 기반 알고리즘을 만든다.

## ⚠️ 가장 중요한 전제 — 기존 백테스트 데이터 재사용 (재수집 금지)
- 새로 가격 데이터를 모으거나 재수집하지 말 것 (시간 너무 날림).
- 로컬 PC의 lassi.db 안에 이미 있는 백테스트 데이터를 활용:
  - backtest_trade_signals : 신호조합·시장국면별 max_gain_pct / max_drawdown_pct / win20 / days_to_peak 등이 이미 계산돼 있음
  - backtest_ticker_summary
- 이 데이터 위에서 비교/평가/알고리즘 생성. (pykrx/yfinance로 매번 재fetch 금지)
- KR/ 폴더의 standalone 백테스트 스크립트들은 개념 검증용이며 가격을 매번 재fetch함 — 본 작업은 반드시 DB 기반으로 재구성할 것.

## 할 일 (우선순위 순)
1. 국면 파악 비교 백테스트 — 여러 국면 판단 기법 vs 봇 기존 8단계(base/market_phase.py). 어느 게 국면을 더 잘 보나.
2. 하락장 대응 비교 — 하락장일 때 각 기법의 매매 결과 vs 기존 프로그램(봇 헤지).
3. 횡보장 / 상승장에서도 동일 비교 — 장세별로 무엇이 최강인지.
4. 모든 기법 총동원 + 경우의 수 비교:
   - ATR(트레일링스톱 / 변동성 타겟팅), 200일선 룰, 골든/데드크로스, 듀얼모멘텀, RSI, MACD, 볼린저밴드, 돈치안/켈트너 채널, ADX, 일목균형표, 상대강도(RS), 거래량 등 — 주식 기법이란 기법 전부.
   - 파라미터 여러 종류(MA 기간, ATR 배수, RSI 임계값 등) + 조합 방식(AND / OR / 투표·가중치) 전수 비교.
5. 위 비교 데이터로 알고리즘 생성 — 장세·종목별 최적 신호 조합.
6. 1순위 / 2순위 등 상황별 대처 방안 — 국면·변동성에 따라 어떤 전략을 우선 쓸지 룰 정립.

## 7. 추가로 해야 할 것 (Claude 제안)
- 과최적화 방지: 워크포워드 / 아웃오브샘플 검증 필수. 기법·파라미터가 많을수록 커브핏(과최적화) 위험이 큼.
- 평가 기준: 단순 수익률만 보지 말고 위험조정수익(샤프, 수익/MDD), 매매횟수(거래비용), 보유%(신호가 너무 빡빡한지)까지 함께 본다.
- 여러 종목 검증: 에스앤에스텍 1개만 보지 말고 보유 종목 전부 + 변동성/시총 다양하게 — 한 종목 운빨 배제.
- 거래비용·세금·슬리피지 현실 반영.
- 봇 8단계의 공정 비교: backtest_trade_signals로 봇 실제 신호 성과를 그대로 끌어와 비교.
- 시장신호 vs 종목신호 구분 (코스피/코스닥 국면 vs 종목 자체 추세).
- 결론은 "감"이 아니라 "숫자"로 — 여태 지켜온 원칙 유지.

## 현재까지 핵심 발견 (에스앤에스텍 101490, 하락장 2021-11 ~ 2022-12)
