## 진행 상황
- [x] A. US 앙상블 게이트 (KR과 통일) — 완료
- [x] B. 통합 연동 검증 (8/8 통과)
- [x] C. 거래비용 — 라이브(_net_profit: 매도세0.18%+수수료0.015%) ✅, US(_net_profit_usd) ✅, 백테스트(cost_pct 양쪽=보수적) ✅. 미세점: 백테스트 cost ~0.42%(의도0.21%)로 양쪽적용=보수적이라 무해. 결론: 비용 정상 반영.
- [x] D. 손절/익절 정렬 — **갭 발견(중요)**: _forecast_block은 백테스트 통계 손절/목표(예 -16%/+33%)를 메시지에 '표시'만 하고, 실제 청산은 ATR기반+고정임계값(check_giveback_stop/RSI익절 등)로 동작 → **표시값 ≠ 실제 청산 불일치**. 권고: 실제 손절선=avg_drawdown 통계, 익절=days_to_peak/target 통계로 정렬해야 "데이터 활용" 완성. 단 라이브 청산로직 변경은 리스크 → 사용자 확정 후 진행.
- [x] E. 앙상블 균형 — 정상: 게이트는 '당일 크로스 신호 발생 + 점수통과' 둘 다 요구 → 점수제 단독보다 선별적이나 0건 아님(대결서 15종목 history 720건). 의도된 선별성(백테스트 진입과 일치). 다만 라이브는 '당일 신호'만 봐서 점수제보다 매수빈도↓ — 이게 곧 +4.5% 고품질의 대가.
- [ ] F. 필터 개선: win10(120일10%달성, 너무 후함) → forward 실수익 기반 게이트 검토
- [ ] G. 코드 정리: 중복/죽은 코드 (단 점수제는 앙상블로 살림 — 삭제X)
