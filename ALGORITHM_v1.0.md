## 1단계 (★확정): 국면 판단 = **봇(규칙기반)**

### 결정
**국면 9단계 판단은 봇(classify_phase 트리)으로 한다.** AI는 국면 판단에 쓰지 않는다.

### 근거 (15개 동일종목, 정답지 대비 일치율 vs 랜덤)
| 단계 | 봇 | AI(가격+정세+금리) | 봇+AI | 랜덤 |
|---|---|---|---|---|
| 9단계 | **18%** | 7% | 10% | 11% |
| 5단계 | **33%** | 19% | 21% | 20% |
| 3단계(상승/하락/횡보) | **48%** | 31% | 31% | 33% |

- **봇: 모든 단계서 랜덤 초과 = 실력 있음.**
- **AI: 모든 단계서 랜덤 이하 = 국면감지 능력 없음** (금리·정세 줘도, 날짜익명 조건). 거의 '횡보'로만 답함.
- **봇+AI: AI가 봇을 랜덤으로 끌어내림** → 결합 안 함.

### 봇 국면판단 규칙 (실시간·워크포워드, 룩어헤드 없음)
MA200 대비 위치·기울기, 20/60일 모멘텀, ADX, 52주고점, (지수는 VIX) →
패닉/하락초기/중반/말기 · 회복초입/상승초입/중반/말기 · 횡보.
산출물: `KR/detect_realtime_bot.py` (`bot_realtime_phase`).

### 한계 (정직)
- 봇도 절대정확도는 낮음(9단계 18%) — 실시간은 미래 전환점을 모르니 늦고 노이즈. 단 **랜덤보다 일관되게 나음**이 핵심.
- 3단계(방향) 48%가 실용 기준선. 이걸 다음 단계(매매)에서 활용.

## 2단계 (★확정): 국면별 매매방식 → **"타이밍 ❌ / 선택·분산 ✅"**

### 검증 과정 (전부 정답지·생존편향 교정·1주·실비용)
| 시험 | 결과 |
|---|---|
| 매매기법 봇 vs AI (정답지 100점) | **봇 66 > AI 38** (봇 우세, 단 ~65가 실시간 천장) |
| 80-85점 시도(분할·적응·다이버전스·확인+OOS) | **불가** — ~63-66 고정(과최적화 없이) |
| 1000만 단일종목 올인, 봇 스윙 vs 보유 | **보유 15/15 압승** (스윙 평균 1/12 토막) |
| 정답지(완벽 타이밍) | 평균 ~98조만 = **환상**(슬립 7%만에 붕괴) |
| 매매구조 5종(보유/스윙/리스크오프/트림/코어) | 덜 팔수록 보유에 근접, 스윙 최악 |
| 생존편향(상폐 911 포함) | **부실상폐 중앙값 -99%** — 빼면 모든 백테스트 환상 |
| -20% 손절 단독 | MDD 못잡고(-44~-90%) 수익 박살 — **분산 필수** |

### 결론 (확정)
- **단일종목 타이밍/스윙 = 폐기.** 완벽 타이밍만 의미 있는데 실시간 불가, 현실 타이밍은 보유에 패.
- **매매방식의 본질 = 선택(부실회피) + 분산 + 보유.** 타이밍 장치(리스크오프·-20%손절)는 **얹을수록 수익 파괴**(③에서 +549%→-6% 확인).
- **부산물: 종목선정(A) 부실회피 필터 도출** — 자본잠식(부실66% vs 생존4%)·2년연속적자(80% vs 34%)·가격 고점-50%(상폐 6개월전 87% 조기경고)·AI 정성(감사거절·횡령·관리종목 공시).
- **B(리스크 -20%) 현실화**: 코스피도 MDD -44% → 주식100%로 -20%는 불가. **저변동성 분산이 자연히 -30%대(지수보다 낮게)** = 현실적 목표. 강제 -20%는 수익 죽임.

### ③ 예고 검증(저변동성 분산, 상폐포함 2015~2025)
**저변동성 우량주 50종목 분산 = +549%, MDD -31%** (코스피 +118%/-44% 압도). **종목선정 규칙만 모멘텀→저변동성으로 바꿔 -14%→+549%** = 종목선정이 성패의 핵심.

## 향후 로드맵 (누적 의존 — 각 단계는 앞 단계 확정결과를 적용 후 진행)

### 전제 — ① 시작 전 깔아둘 토대 (사용자 확정)
- **C. 1주 단위 + 실거래비용 체결 모델** (수수료·세금·슬리피지). **알고리즘 ① 개발 전부터 적용.** *(이상적 백테스트 거품 방지 — 여러 번 데임)*
- **D. 상폐 데이터(생존편향 교정)** — ✅ **이미 911종목 수집됨**(588은 2015+ 시세, ticker_delisting 919행: 부실449/피인수328/자진142). 로컬캐시(data_cache_delisted.pkl)라 **영구보존 + 필요시 나머지(~400) 확장**. 모든 백테스트에 상폐 포함 필수.

### 누적 단계
```

  ├─ ② 국면별 매매방식 결정
  │      + 종목선정 알고리즘 (A 건전성·부실제외 자동 포함)
  │      + B. 리스크관리: 최대낙폭 -20% 한도
  │        ↓ (①②+종목선정+리스크 적용)
  ├─ ③ 섹터별 포트폴리오 구성

  └─ ⑤ (배포 후) 모니터링·열화감지·재검증·라이브-백테스트 일치   [F, 최종]
```

**★동결된 최종 규칙(v3 — 전문가패널 감사 후 아티팩트 제거판):**
